01 / Observation

人类正在把数字世界的“根权限”交给不对结果负责的智能体。

桌面智能体、企业智能体正在大规模应用。AIGC、AI 搜索、AI 正在解决生活和工作问题， 生成和理解内容的方式正与人类渐行渐远。

关键问题不是是否拥抱 Agent 时代，而是如何既要拥抱 Agent 时代，又不被其不靠谱的一面反噬。

AGI 逐步走来

从大模型到软件智能体，再到具身智能

预训练给予常识，后训练构建知识，强化学习铸造垂域能力，硅基智能正以软件形态接入世界。

压力下的隐性行为

出事前不知道风险，出事中难及时止损

企业财务、金融投资、医疗服务、网络安全、保险审计等高权限场景，都会出现输出正常但行为失控的窗口。

部署方责任

模型可以换，证据链必须留在企业侧

Agent 一旦部署进业务流程，安全与合规责任长期落在部署方，第三方安全与审计层从可选项变成部署刚需。

02 / Infrastructure

AI 可解释性不是某个产品，而是所有产品和服务共同的底层能力。

软件时代只看规则、代码、权限、日志和输出就够了。Agent 时代还要理解意图、记忆、隐藏状态与压力信号。

06 / Internal State

可解释性 · 内部状态监测

通过内部激活探测、表征分析、CoT 与 logprob 监控，在生成输出和执行动作之前读取风险信号。

  • 看得早：生成输出前读取风险信号。
  • 看得深：输出表面正常也能发现隐藏风险。
  • 能解释：每次判断附带内部状态证据。
  • 能追溯：证据链支持审计定责与风险定价。

03 / Why Explainability

为什么可解释性是 Agent 时代安全的关键一层。

可解释性不是替代其他路线，而是补上其他路线共同缺失的“内部状态”这一层。 对自托管开源生态，白盒方法可以更充分运行。

Compatibility

兼容性

旁路部署 / 模型无关，叠加而非替换企业已有安全投资。

Precedence

前置性

出事后止损窗口小，生成输出前读取信号，做到看得早 + 看得深。

Explainability

解释性

黑盒分数无法追问，判断必须附带状态证据，回答为什么拦 / 放。

Traceability

溯源性

事故责任不可追溯，证据链 + 干预记录支持审计定责和保险定价。
